## 처음으로 레포지토리를 직접 만들었다

회사에서 GitHub을 아예 안 써본 건 아니다. 콘텐츠 관리 작업을 하면서 파일을 수정하고, 커밋하고, 푸쉬하고, PR을 올리는 일을 해왔다. 근데 항상 **누군가 이미 만들어놓은 레포지토리**에 들어가서 하는 일이었다.

처음부터 레포를 만들고, 코드를 올리고, 거기서 배포까지 연결하는 건 오늘이 처음이었다.

생각보다 별 게 아니었는데, 막상 해보니 신기했다.

---

## 1단계: 로컬에서 콘텐츠 만들기

블로그 포스트는 `posts/` 폴더 안에 마크다운 파일로 관리한다. 파일 이름이 그대로 URL slug가 된다.

```
posts/
├── ai-vibe-coding-first-day.md
├── weather-app.md
└── first-project-profile-page.md
```

새 글을 쓰려면 그냥 파일 하나 추가하면 된다. frontmatter에 제목, 날짜, 태그를 넣고, 아래에 마크다운으로 본문을 쓰면 끝이다.

```markdown
---
title: "글 제목"
date: "2026-04-07"
tags: ["태그"]
published: true
---

본문 내용
```

`npm run dev`를 켜두면 파일 저장하자마자 브라우저에서 바로 확인할 수 있다.

![로컬 개발 서버에서 확인하는 블로그 화면](/images/posts/blog-local.png)

---

## 2단계: GitHub에 푸쉬

로컬에서 잘 보이면 GitHub에 올린다.

```bash
git add .
git commit -m "post: 새 글 추가"
git push origin main
```

회사에서도 이 흐름은 똑같이 해봤다. 근데 그때는 레포가 이미 있었고, 브랜치 규칙도 정해져 있었고, PR 올리면 리뷰어가 있었다. 여기선 내가 직접 레포를 만들고, 브랜치 전략도 내가 정하고, 머지도 내가 한다.

혼자 하는 거라 단순하긴 한데 — 전체 흐름이 내 손 안에 있다는 게 느낌이 달랐다.

![GitHub에 푸쉬된 커밋 화면](/images/posts/blog-github.png)

---

## 3단계: Vercel로 배포

GitHub 레포를 Vercel에 연결해두면, `main` 브랜치에 푸쉬할 때마다 자동으로 배포가 돌아간다.

처음 연결할 때만 Vercel 대시보드에서 레포 가져오기 → Deploy 한 번 하면 된다. 이후엔 푸쉬만 하면 알아서 빌드되고 배포된다.

Next.js 프로젝트라 별도 설정(`vercel.json` 같은 것)도 필요 없었다. 환경 변수도 없었으니 그냥 연결하자마자 바로 떴다.

![Vercel 배포 완료 화면](/images/posts/blog-vercel.png)

배포가 끝나면 `https://[프로젝트명].vercel.app` 같은 URL이 생긴다. 브라우저에서 그 주소로 들어가면 실제로 올라간 블로그가 보인다.

---

## 전체 흐름 정리

1. **로컬에서 `.md` 파일 작성** — `posts/` 폴더에 마크다운 파일 추가
2. **`npm run dev`로 확인** — 저장하자마자 브라우저에서 바로 미리보기
3. **`git push origin main`** — GitHub에 커밋 푸쉬
4. **Vercel 자동 빌드 & 배포** — 푸쉬 감지 → 빌드 → 배포까지 자동
5. **브라우저에서 접속** — 발급된 URL로 누구나 접근 가능

글 하나 올리는 데 코드 배포 파이프라인이 통째로 돌아가는 구조다. 처음엔 좀 과한 거 아닌가 싶었는데, 막상 익숙해지니까 오히려 편하다. 파일 저장하고 푸쉬만 하면 알아서 다 처리되니까.

---

## 아직 완전히 이해된 건 아닌데

솔직히 말하면, 이 과정을 처음부터 끝까지 다 이해한 건 아니다.

Vercel이 빌드를 어떻게 하는지, Next.js가 정적 파일을 어떻게 다루는지, CI/CD가 정확히 어떤 식으로 돌아가는지 — 아직 겉핥기 수준이다.

근데 회사에서 남이 만든 레포에 커밋 날리던 것과, 처음부터 직접 만들어서 배포까지 연결한 것 사이의 간격은 꽤 크게 느껴졌다. 이해도가 아니라 경험의 차이랄까.

계속 쓰다 보면 자연히 채워지겠지 싶다.
