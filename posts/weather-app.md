## 이번엔 뭔가 기능이 있는 걸 만들고 싶었다

자기소개 페이지를 만들고 1주일 후였다. 정적인 HTML에서 한 발 더 나아가, 실제로 동작하는 걸 만들어보고 싶었다. 날씨 앱으로 정한 이유는 단순하다 — API 연동이 뭔지 궁금했고, 지도 UI가 있으면 재밌을 것 같았다.

## 만든 기능들

### 한국 행정구역 지도

SVG로 그린 한국 지도. 서울, 부산, 제주 등 17개 행정구역을 클릭하면 해당 지역 날씨를 바로 불러온다. 지도에서 직접 선택하는 게 생각보다 훨씬 편했다.

### 실시간 날씨 정보

OpenWeatherMap API를 연동했다. 지역을 선택하면:

- 현재 기온 / 체감 온도
- 날씨 상태 (맑음, 흐림, 비...)
- 습도, 풍속
- 날씨 아이콘

### 주간 예보

오늘부터 5일치 예보를 카드 형태로 보여준다. 요일별 최고/최저 기온과 날씨 아이콘.

### 낮/밤 모드

일출·일몰 시간을 기준으로 자동으로 배경색이 바뀐다. 낮엔 밝고, 밤엔 어두운 톤으로.

### 검색 기능

지도 클릭 외에도 텍스트로 검색할 수 있다. 입력하면 행정구역 드롭다운이 뜨고, 선택하면 날씨 로딩.

## 처음 겪은 것들

**API 키 관리** — `.env` 파일에 키를 넣고 `import.meta.env`로 읽는 방식. 처음엔 왜 `process.env`가 안 되는지 몰랐는데 Vite는 방식이 다르다는 걸 알게 됐다.

**비동기 처리** — `useEffect` 안에서 `fetch`를 쓰는 패턴. async/await이 어떻게 동작하는지 코드를 보면서 감을 잡았다.

**날씨 상태별 배경** — 맑음, 흐림, 비, 눈, 태풍마다 다른 색상을 객체로 관리했다. 조건에 따라 스타일을 동적으로 바꾸는 게 이렇게 하는 거구나 싶었다.

## 아쉬운 점

지도 SVG를 직접 그리는 과정에서 행정구역 경계가 정확하지 않다. 대략적인 모양이라 실제 지도랑 다른 부분이 있다. 다음엔 GeoJSON 데이터를 활용해봐야겠다.

## 소감

자기소개 페이지랑은 달리 이번엔 뭔가 "작동하는 것"을 만들었다는 느낌이 훨씬 강했다. API 연동, 상태 관리, 조건부 렌더링 — 이런 단어들이 코드를 보면서 자연스럽게 이해되기 시작했다.

바이브코딩으로 만든 두 번째 작품. 만든 지 1주일이 됐는데 아직도 가끔 열어서 날씨 확인한다. 점점 욕심이 생긴다.
