## 바이브코딩이 뭔데?

"바이브코딩(Vibe Coding)"이라는 말을 처음 들었을 때는 반신반의했다. AI한테 말로 설명하면 코드가 나온다는 건데, 그게 실제로 쓸 만할까?

오늘 직접 해봤다. 결론부터 말하면 — 진짜 된다.

## 오늘 한 일

오늘 목표는 단순했다. 개인 기술 블로그 하나 만들기.

평소였으면 보일러플레이트 세팅, 디렉토리 구조 고민, 마크다운 파싱 라이브러리 찾기, 타입 정의... 이것만 해도 반나절은 썼을 것이다. 그런데 오늘은 달랐다.

Claude에게 말했다:

> "Next.js 15 + TypeScript + Tailwind로 개인 기술 블로그 만들어줘. 마크다운 파일로 포스트 관리하는 구조로."

그리고 잠깐 기다렸더니 다음이 갖춰져 있었다:

- `src/lib/posts.ts` — `gray-matter`로 frontmatter 파싱, `remark`로 HTML 변환
- `src/types/post.ts` — `Post`, `PostMeta` 타입
- 홈 / 블로그 목록 / 포스트 상세 / About 페이지
- 다크모드 대응 레이아웃

`npm run build`를 돌렸더니 오류 없이 통과했다.

![블로그 화면](/images/posts/blog-screenshot.png)

## 느낀 점

### 좋았던 것

코드를 처음부터 짜는 시간이 거의 없었다. 내가 한 일은 주로 **검토하고 판단하는 것**이었다. "이 구조가 맞나?", "이 컨벤션이 내 스타일인가?" 같은 질문들.

지루한 보일러플레이트 작업에서 해방된 느낌이 들었다. 그 시간을 아껴서 실제로 뭘 만들지 생각하는 데 쓸 수 있었다.

### 아직 사람이 해야 하는 것

AI가 만들어준 코드를 그냥 쓰면 안 된다. 읽어야 한다. 오늘도 생성된 코드를 한 줄씩 훑으면서 "왜 이렇게 했지?", "이게 최선인가?" 를 계속 생각했다.

결국 바이브코딩은 개발 속도를 높여주는 도구일 뿐이다. 판단과 책임은 여전히 사람 몫이다.

## 앞으로

이 블로그도 바이브코딩으로 만들었으니, 앞으로 기능을 추가하거나 글을 쓸 때도 같은 방식으로 해볼 생각이다. 어디까지 가능한지 직접 써보면서 기록해나갈 것이다.

디자인은 솔직히 아직 많이 부족하다. 시키는 대로 만들었더니 기본 중의 기본으로 나왔는데, 그냥 이 상태로 두고 계속 써나가면서 조금씩 고쳐갈 생각이다. 완성된 블로그보다 성장하는 블로그가 더 재밌을 것 같다.
